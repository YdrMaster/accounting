账户视图扁平网格 V2 设计

本文件用真实 HTML/CSS 渲染目标效果。所有配色与关键样式均以内联方式写在元素上，避免外部样式干扰，保证浏览器中看到的即最终效果。

1. 核心布局算法：compile

参考 .ignore/show-layout/src/main.rs 中的实现。该函数把多级树表示转换成每行固定宽度的渲染格式。

use std::fmt::Write;

/// 这个函数将多级树表示转换成渲染格式。
///
/// `input` 是一个多级树表示字符串，表示一个树，其每个级别的兄弟分支（用 `,` 隔开）中有 0 个或 1 个有子树。
/// 名字前面的 `+` 表示有子树的分支。`;` 分隔下一级子树。
///
/// `width` 是渲染宽度，表示渲染格式每行的元素数量。
fn compile(input: &str, width: usize) -> String {
    let mut levels = input.split(';').map(|s| s.split(',')).collect::<Vec<_>>();
    let mut depth = 0;
    let mut ans = String::new();

    'outer: loop {
        let mut pointer = None;
        let source = &mut levels[depth];

        for i in 0..width {
            let Some(mut item) = source.next() else {
                if i == 0 {
                    if depth == 0 {
                        return ans;
                    }
                    depth -= 1;
                    continue 'outer;
                }
                for _ in i..width {
                    write!(ans, ",").unwrap()
                }
                break;
            };

            if let Some(item_) = item.strip_prefix('+') {
                pointer = Some(i);
                item = item_;
                depth += 1
            }

            write!(ans, "{item},").unwrap()
        }

        writeln!(ans).unwrap();
        if let Some(pointer) = pointer {
            for _ in 0..pointer {
                write!(ans, "_").unwrap()
            }
            write!(ans, "^").unwrap();
            for _ in pointer + 1..width {
                write!(ans, "_").unwrap()
            }
            writeln!(ans).unwrap()
        }
    }
}

3. 折叠条件变更

仅当新选中卡片 不在 当前 expandedPath 上时才折叠；在路径上时只切换选中态，子树保持展开。

假设当前展开路径为 [C, C3, C31]，对应树：C → C3 → C31（C311/C312 已展开）。

点击卡片 是否在 expandedPath 上 行为 折叠后路径
C3 是 仅切换选中态，不折叠 [C, C3, C31] 不变，C311/C312 保持
C32 否 折叠 C31 子树，展开 C32 子树 [C, C3, C32]，C311/C312 被折叠
C 是 仅切换选中态，不折叠 [C, C3, C31] 不变
D 否 折叠 C 子树，展开 D 子树 [D]

4. 层叠高亮链（包裹式纸片叠放）

保持 A/B 方案的嵌套包裹结构，但去掉缩进：每一级 .children-container 都和父级容器等宽。层叠容器去掉左右 padding（padding: 0.5rem 0），使所有深度的卡片等宽；上下用负 margin 使背景上下边缘与父级重叠；::before 伪元素向左右各扩展 0.5rem（left: -0.5rem; right: -0.5rem），形成高亮区域并与卡片保持间距。所有层级的高亮左右边缘相互对齐并重叠，半透明背景相互叠加，像几张半透明的纸压在一起。高亮做在伪元素背景上，卡片颜色保持统一。

5. 实现要点

  • compileRows 已按上述算法实现，保持每行固定列数、左对齐、列跨深度对齐。
  • buildRowTree 把扁平的 GridRow 还原为嵌套结构，继续用 AccountRowGroup 递归渲染。
  • AccountRowGroup 中的 .children-container 保持 subgrid 与默认 padding/margin（保证列对齐）；层叠容器 .stacked-children 去掉左右 padding（padding: 0.5rem 0），使所有深度的卡片等宽；上下使用负 margin 使背景上下边缘与父级重叠；::before 伪元素向左右各扩展 0.5rem（left: -0.5rem; right: -0.5rem），形成高亮区域并与卡片保持间距；所有层级的高亮左右边缘相互对齐并重叠，半透明背景相互叠加，形成纸片叠放效果；背景色按深度使用 HSV 渐变。
  • 折叠条件：在 AccountsView.handleAccountClick 中，先判断 clickedId 是否在 expandedPath 中。若不在，才重新构建路径；若在，仅更新 selectedAccountId。

6. 最终方案

  • 配色：方案 C（HSV 深度渐变）
  • 布局：所有卡片等宽，所有高亮区域等宽且与卡片对齐，不做缩进，纸片叠放效果
  • 折叠条件：当且仅当新选中卡片不在当前展开路径上时才折叠
  • 层叠高亮：每级展开路径加等宽、全宽、半透明背景框，多层叠加，卡片颜色统一
