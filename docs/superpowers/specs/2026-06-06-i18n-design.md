# 多语言支持（i18n）设计方案

## 背景与目标

当前项目所有用户-facing 的文本都是硬编码的中文：
- `AccountingError` 的 `#[error("...")]` 消息
- CLI `print_line` 中的操作提示
- 数据库 seed 中的系统账户名称、商品名称、标签描述

目标：**至少支持简体中文（zh-CN）和英文（en）**，用户可通过 `--lang` 参数或 `LANG` 环境变量切换语言。

## 设计原则

1. **编译期嵌入**：语言文件在编译时打包进二进制，零运行时 IO
2. **分层隔离**：核心错误信息在 `accounting` crate 翻译，CLI 提示在 `accounting-cli` crate 翻译
3. **最小侵入**：不改动现有业务逻辑，仅替换字符串输出层
4. **英文兜底**：任何缺失的翻译自动回退到英文

## 技术选型

**`rust-i18n` v3.x**
- 编译期代码生成，YAML 语言文件嵌入二进制
- 全局 `t!` 宏 + `set_locale` 运行时切换
- 支持 `std::fmt` 语法的参数化翻译（v3.1.0+）
- 自动 fallback 链：`zh-CN` → `zh` → `en`

**`src/error.rs` 改造：**

删除所有 `#[error("...")]` 属性，改为手动实现 `Display`：

```rust
use rust_i18n::t;

#[derive(Debug, Clone, PartialEq)]
pub enum AccountingError {
    InvalidTransaction(String),
    AccountNotFound(String),
    AccountNotEmpty(String),
    CommodityNotFound(String),
    AccountAlreadyExists(String),
    InvalidDate(String),
    DatabaseError(String),
    DbNotInitialized,
    DbAlreadyExists,
    Unknown(String),
}

impl std::fmt::Display for AccountingError {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        match self {
            Self::InvalidTransaction(msg) => {
                write!(f, "{}", t!("invalid_transaction", msg = msg))
            }
            Self::AccountNotFound(name) => {
                write!(f, "{}", t!("account_not_found", name = name))
            }
            Self::AccountNotEmpty(name) => {
                write!(f, "{}", t!("account_not_empty", name = name))
            }
            Self::CommodityNotFound(name) => {
                write!(f, "{}", t!("commodity_not_found", name = name))
            }
            Self::AccountAlreadyExists(name) => {
                write!(f, "{}", t!("account_already_exists", name = name))
            }
            Self::InvalidDate(msg) => {
                write!(f, "{}", t!("invalid_date", msg = msg))
            }
            Self::DatabaseError(msg) => {
                write!(f, "{}", t!("database_error", msg = msg))
            }
            Self::DbNotInitialized => {
                write!(f, "{}", t!("db_not_initialized"))
            }
            Self::DbAlreadyExists => {
                write!(f, "{}", t!("db_already_exists"))
            }
            Self::Unknown(msg) => {
                write!(f, "{}", t!("unknown_error", msg = msg))
            }
        }
    }
}

impl std::error::Error for AccountingError {}
```

> **为什么不用 `#[error]` + `t!`？** `#[error("{}", t!(...))]` 涉及两个过程宏的展开顺序，有编译风险。手动实现 `Display` 更可靠，且 `thiserror` 的 `#[derive(Error)]` 可以检测到手动的 `Display` 实现并直接使用。

**`src/main.rs` 改造：**

添加 `--lang` 全局参数和语言检测逻辑：

```rust
#[derive(Parser)]
#[command(version, about)]
pub struct Cli {
    pub db: PathBuf,
    #[arg(long, value_enum, default_value = "table")]
    pub format: OutputFormat,
    /// 语言（如 zh-CN、en）
    #[arg(long)]
    pub lang: Option<String>,
    #[command(subcommand)]
    pub command: Commands,
}

fn detect_language(cli_lang: Option<&str>) -> String {
    if let Some(lang) = cli_lang {
        return lang.to_string();
    }
    if let Ok(lang) = std::env::var("LANG") {
        return lang.split('.').next().unwrap_or("en").to_string();
    }
    "en".to_string()
}

#[tokio::main]
async fn main() {
    let cli = Cli::parse();
    let lang = detect_language(cli.lang.as_deref());
    rust_i18n::set_locale(&lang);
    // ...
}
```

### 3. accounting-sql crate（内置数据 i18n）

系统账户名称、商品名称、标签描述在数据库 seed 中硬编码。方案：**初始化时根据语言参数选择 seed SQL**。

**`src/schema.rs` 改造：**

将 seed SQL 拆分为语言相关的常量：

```rust
const SEED_ACCOUNTS_EN: &str = r#"
INSERT OR IGNORE INTO accounts (full_name, account_type, parent_id, is_system) VALUES
('Equity:OpeningBalances', 3, NULL, 1),
('Income:Uncategorized', 4, NULL, 1),
('Expenses:Uncategorized', 5, NULL, 1),
('Expenses:Fees', 5, NULL, 1),
('Expenses:Discounts', 5, NULL, 1),
('Expenses:InstallmentFees', 5, NULL, 1),
('Assets:Cashback', 1, NULL, 1);
"#;

const SEED_ACCOUNTS_ZH: &str = r#"
INSERT OR IGNORE INTO accounts (full_name, account_type, parent_id, is_system) VALUES
('权益:期初余额', 3, NULL, 1),
('收入:未分类', 4, NULL, 1),
('费用:未分类', 5, NULL, 1),
('费用:手续费', 5, NULL, 1),
('费用:折扣', 5, NULL, 1),
('费用:分期手续费', 5, NULL, 1),

const SEED_COMMODITIES: &str = r#"
INSERT OR IGNORE INTO commodities (symbol, name, precision) VALUES ('CNY', '人民币', 2);
"#;

const SEED_TAGS_EN: &str = r#"
INSERT OR IGNORE INTO tags (name, description, is_system) VALUES
```

`insert_seed_data` 函数接收语言参数：

```rust
pub fn insert_seed_data(conn: &Connection, lang: &str) -> Result<(), DbError> {
    let accounts_sql = if lang.starts_with("zh") {
        SEED_ACCOUNTS_ZH
    } else {
        SEED_ACCOUNTS_EN
    };
    let tags_sql = if lang.starts_with("zh") {
        SEED_TAGS_ZH
    } else {
        SEED_TAGS_EN
    };
    conn.execute_batch(&format!("{}{}{}{}", 
        SEED_COMMODITIES, accounts_sql, tags_sql
    ))?;
    Ok(())
}
```

**`initialize` 命令改造：**

```rust
async fn initialize(
    db_path: &std::path::Path,
    lang: &str,
) -> Result<(), AccountingError> {
    if db_path.exists() {
        return Err(AccountingError::DbAlreadyExists);
    }
    let db = SqliteDatabase::open(db_path.to_str().unwrap())
        .map_err(|e| AccountingError::Unknown(e.to_string()))?;
    db.initialize(lang)
        .map_err(|e| AccountingError::Unknown(e.to_string()))?;
    println!("{}", t!("db_initialized"));
    Ok(())
}
```

### 4. 语言检测优先级

```
1. --lang 命令行参数（最高优先级）
2. LANG 环境变量（如 zh_CN.UTF-8 → zh-CN）
3. 系统默认 locale
4. en（英文兜底）
```

### 5. 与现有 `fmt::Display` 的集成

`main.rs` 中 `eprintln!("错误: {}", e)` 已经使用 `Display` 格式。`AccountingError` 手动实现 `Display` 后，会自动调用 `t!` 宏返回翻译后的文本，无需改动 `main.rs` 的错误输出代码。

## 测试策略

1. **错误翻译测试**：设置 `rust_i18n::set_locale("en")`，验证 `AccountingError::DbNotInitialized.to_string()` 返回英文
2. **错误翻译测试**：设置 `rust_i18n::set_locale("zh-CN")`，验证返回中文
3. **缺失翻译回退**：设置不存在的 locale（如 "fr"），验证回退到英文
4. **参数化翻译测试**：验证 `t!("tx_created", id = 42)` 正确替换参数
5. **Seed 数据测试**：验证 `zh` 和 `en` 两种语言下系统账户名称正确
