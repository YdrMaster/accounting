# 交易列表概览布局设计

## 背景
当前交易列表概览为单行布局（时间 + 备注 + 记账人 + 金额），信息密度低，无法一眼看出交易涉及的账户。

## 目标
改为 2 行 3 栏布局，在概览中直接展示收支账户和资产账户，提升可读性。

## 设计

### 布局结构（2 行 × 3 栏）

```
┌─────────────┬────────────────────────────┬─────────────┐
│  记账人     │  收入账户 支出账户          │   ¥金额     │
│  时间       │  备注                       │   资产账户  │
└─────────────┴────────────────────────────┴─────────────┘
```

- **栏1（左）**：记账人 + 时间，垂直居中，居中对齐
- **栏2（中）**：收入/支出账户（第一行）+ 备注（第二行），左对齐
- **栏3（右）**：金额（第一行）+ 资产账户（第二行），右对齐

### 账户显示规则

1. **只显示账户最后一级**（如 `Expenses:Food` → `Food`）
2. **账户之间用空格隔开**
3. **收入账户显示为绿色**，**支出账户显示为红色**
4. **资产账户显示为灰色**

### 金额颜色规则

| 场景 | 金额颜色 |
|------|---------|
| 只有支出，没有收入 | 红色 `#f5222d` |
| 只有收入，没有支出 | 绿色 `#52c41a` |
| 既有收入又有支出 | 灰色（明亮 `#666`，暗黑 `#aaa`） |

### 资产文字颜色

- 金额下方资产账户文字**始终显示为灰色**
- 灰色与非纯收支时的金额颜色一致（明亮 `#666`，暗黑 `#aaa`）

## 实现范围

仅修改 `accounting-web/src/components/TransactionDetail.vue`：
- 模板结构改为 3 栏 flex/grid
- 新增 computed：支出账户列表、收入账户列表、资产账户列表、金额颜色判断
- 引入 `useAccountStore` 以查询账户类型
- 适配暗色主题全局样式（`App.vue` 中补充 `.transaction-amount` 暗色颜色规则）
