# 账户管理页面 — 卡片手风琴重设计

## 背景

当前账户页面使用 `a-tree` 树状组件展示所有层级，在宽屏下内容集中在左侧窄条，空间利用率低。需要改为卡片阵列 + 手风琴展开的方式。

## 设计目标

1. 顶级账户分类（资产/负债/收入/支出/权益）使用页面选项卡切换
2. 子账户以 flex-wrap 卡片网格展示，适应宽屏
3. 卡片支持选中（查看详情）和展开（查看子账户）两种交互，**互斥**：同时只能有一个账户被选中或展开
4. 展开后子账户卡片插在原卡片下方，同级卡片被下推
5. 禁止添加顶级账户，仅允许添加子账户
6. 支持账户重命名、关闭、重开，新建和重命名时校验同名

## 总体方案

### 1. 布局结构

```plaintext
┌─────────────────────────────────────────────────────┐
│ [资产] [负债] [收入] [支出] [权益]   ← a-tabs 固定  │
├─────────────────────────────────────────────────────┤
│                  卡片网格区域                       │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────────┐            │
│  │ Card │ │ Card │ │ Card │ │ + 添加   │            │
│  └──────┘ └──────┘ └──────┘ └──────────┘            │
│  ┌──────┐ ┌──────┐   ← 展开后插入                   │
│  │ Sub  │ │ +添加│                                  │
│  └──────┘ └──────┘                                  │
├─────────────────────────────────────────────────────┤
│              详情面板（选中后显示）                 │
└─────────────────────────────────────────────────────┘
```

### 2. 交互模型

**互斥规则：** 同时只能有一个账户处于"选中或展开"状态。

| 操作 | 结果 |
|------|------|
| 选中 A | A 高亮；如有其他选中/展开的账户，清除 |
| 展开 A | A 展开子账户；如有其他展开的账户，那组收起；A 的选中状态不受影响 |
| 展开已选中的 A | A 展开，选中保持 |
| 选中已展开的 A | A 选中，展开保持 |
| 选中 B（A 已展开） | A 收起，B 选中 |
| 展开 B（A 已选中） | A 取消选中，B 展开 |

- 有子账户：显示子账户数量 + ▼ 箭头
- 无子账户：显示 `+` 添加按钮，点击即展开并进入添加状态
- 拖拽排序：同层级卡片可拖拽调整顺序，拖拽结束后调用 API 持久化

### 3. 卡片状态（互斥模型）

同时仅一个账户处于"选中或展开"状态，简化状态管理：

```plaintext
正常（未选中，未展开）：
┌──────────────┐
│ Bank      ▼  │   有子账户：显示 ▼ 箭头
│ 2 个子账户   │
└──────────────┘
┌──────────────┐
│ Cash      +  │   无子账户：显示 + 按钮（点击添加并展开）
│ 添加子账户   │
└──────────────┘

选中并展开（最常见的激活状态）：
┌══════════════┐
║ Bank      ▲  ║   ← 选中框 + 展开
║ 2 个子账户   ║
╠══════════════╣   ← 子账户（略微缩进）
║ ┌──────────┐ ║
║ │ Savings  │ ║
║ └──────────┘ ║
║ ┌──────────┐ ║
║ │ + 添加   │ ║
║ └──────────┘ ║
└══════════════┘

仅选中（无子账户，或手动展开前）：
┌══════════════┐
║ Bank      ▼  ║
║ 2 个子账户   ║
└══════════════┘
```

- 切换 tab 重置所有状态

### 4. 添加账户

- **有子账户的卡片**：展开后末尾出现 `+ 添加子账户` 卡片，点击弹出内联输入框
- **无子账户的卡片**：卡片上直接显示 `+` 添加按钮，点击后自动展开，子账户区域预置一个已进入添加状态的输入行（输入框已聚焦）
- 输入账户名（只需末级名称），后端自动拼接完整路径
- 确认前检查同层级无同名，确认后调用 `createAccount` API 刷新当前 tab

### 5. 详情面板与操作

选中卡片后，网格下方显示详情面板：

- 完整名称（可点击编辑图标进入内联编辑模式）
- 账户类型（只读）
- 所有者（仅 Asset 类型，checkbox 多选）
- 系统账户标志（橙色 tag）
- 操作按钮：
  - **重命名**：内联编辑名称，校验同层级无同名
  - **关闭账户**：Asset/Liability 需余额为零
  - **重新开启**：已关闭的账户可重开

所有新建和重命名操作需检查目标层级无同名账户（前端批量对比）。

### 6. 选项卡

- 使用 `a-tabs` 组件，5 个固定 tab：资产/负债/收入/支出/权益
- Tab 不可动态添加
- 切换 tab 时重置展开状态和选中状态
- 每个 tab 内容区域独立滚动

## 改动清单

| 文件 | 改动 |
|------|------|
| `accounting-web/src/views/AccountTree.vue` | 整体替换：tabs + 卡片网格 + 手风琴 + 详情面板 + 拖拽排序 |

## 组件结构

```plaintext
AccountTree.vue
├── a-tabs (5 个固定 tab)
│   └── a-tab-pane (×5)
│       └── CardGrid
│           ├── AccountCard (正常卡片)
│           │   ├── 点击选中 → emit select

│           │       ├── AccountCard (递归)

```

## 范围说明

不涉及：

- 余额显示
- 账户删除（物理删除）

## 验证规则

1. 顶级 tab 不可添加/重命名账户
2. 子账户创建/重命名时检查同层级无同名（前端对比 accountStore.accounts）
3. 关闭 Asset/Liability 需余额为零，Income/Expense 无限制
4. 互斥规则：同时只能有一个账户被选中或展开；切换 tab 重置
5. 暗色主题正常显示
6. 重命名成功/失败有 toast 提示
7. 关闭/重开成功刷新 tab 数据
8. 拖拽排序结束后 position 持久化到后端，页面刷新后保持顺序
