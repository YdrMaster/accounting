# account-page

账户管理页面——按根账户类型分组展示账户卡片，支持选中、展开子账户、抽屉式编辑。

## Requirements

### Requirement: 账户卡片网格展示
系统 SHALL 在账户页面按根账户类型分 5 栏竖排展示（资产、收入、支出、权益、导入），每栏内账户以卡片网格展示，卡片只显示账户名称。

#### Scenario: 加载账户列表
- **WHEN** 用户切换到账户页面
- **THEN** 系统从 API 获取所有账户，按根账户类型分栏展示

#### Scenario: 根账户作为栏目标题
- **WHEN** 账户页面渲染
- **THEN** 每栏顶部显示根账户类型名称（资产、收入、支出、权益、导入），根账户不可点击

#### Scenario: 账户卡片只显示名称
- **WHEN** 账户卡片渲染
- **THEN** 卡片上只显示账户名称，不显示余额或其他字段

### Requirement: 选中账户
系统 SHALL 支持点击账户卡片选中账户。

#### Scenario: 点击账户卡片
- **WHEN** 用户点击一个账户卡片
- **THEN** 该账户被选中，从页面底部拉起抽屉显示账户详情和编辑控件

#### Scenario: 点击已选中的账户
- **WHEN** 用户点击当前已选中的账户卡片
- **THEN** 取消选中，抽屉关闭

### Requirement: 展开子账户
系统 SHALL 在选中账户时展开其直接子账户，子账户以高亮行形式显示在父账户卡片下方。

#### Scenario: 选中有子账户的账户
- **WHEN** 用户选中一个有子账户的账户
- **THEN** 该账户的直接子账户以高亮行形式展开显示在父账户下方

#### Scenario: 选中无子账户的账户
- **WHEN** 用户选中一个没有子账户的账户
- **THEN** 不展开任何子账户行

### Requirement: 选中与展开互斥
系统 SHALL 保证选中和展开的互斥逻辑：取消选中时，只有当被取消选中的账户的子树中不包含新选中项时才折叠。

#### Scenario: 选中父账户后选中子账户
- **WHEN** 用户先选中账户 A（展开子账户 B、C），再选中子账户 B（展开 D）
- **THEN** B 被选中并展开 D，A 不再被选中但不折叠（B 在 A 的子树内，折叠 A 会让 B 不可见）

#### Scenario: 同级账户切换
- **WHEN** 用户选中账户 B（展开 D），再选中同级账户 C
- **THEN** C 被选中，B 不再被选中且 D 被折叠（C 和 B 同级，折叠 B 不影响 C 可见性）

#### Scenario: 点击外部取消选中
- **WHEN** 用户点击账户卡片以外的区域
- **THEN** 所有账户取消选中，所有子账户行折叠，抽屉关闭

### Requirement: 抽屉式编辑面板
系统 SHALL 在选中账户时从页面底部拉起抽屉，显示账户详情和 inline 编辑控件。

#### Scenario: 抽屉显示账户信息
- **WHEN** 用户选中一个账户
- **THEN** 抽屉显示账户名称、账单日、还款日、所有者、关闭状态

#### Scenario: 编辑账户名称
- **WHEN** 用户在抽屉中修改账户名称并确认
- **THEN** 系统调用 rename API 更新账户名称

#### Scenario: 编辑账单日和还款日
- **WHEN** 用户在抽屉中修改账单日或还款日并确认
- **THEN** 系统更新对应字段

#### Scenario: 编辑账户所有者
- **WHEN** 用户在抽屉中修改账户所有者并确认
- **THEN** 系统调用 set_owner API 更新所有者

#### Scenario: 关闭账户
- **WHEN** 用户在抽屉中点击关闭账户
- **THEN** 系统调用 close API 关闭账户，卡片显示关闭状态

#### Scenario: 重开账户
- **WHEN** 用户在抽屉中对已关闭账户点击重开
- **THEN** 系统调用 open API 重开账户

#### Scenario: 删除账户
- **WHEN** 用户在抽屉中点击删除账户
- **THEN** 系统调用 delete API 删除账户（根账户和系统账户不可删除）

#### Scenario: 根账户不可编辑
- **WHEN** 用户选中的是根账户
- **THEN** 抽屉中不显示编辑控件，只显示账户信息
