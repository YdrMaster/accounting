# account-page

## Purpose

账户管理页面——按根账户类型分组展示账户卡片，支持选中、展开子账户、抽屉式编辑。

## Requirements

### Requirement: 账户卡片网格展示
系统 SHALL 在账户页面按根账户类型分 4 栏竖排展示（资产、收入、支出、权益），每栏内账户以卡片网格展示，卡片只显示账户名称。

#### Scenario: 加载账户列表
- **WHEN** 用户切换到账户页面
- **THEN** 系统从 API 获取所有账户，按根账户类型分栏展示

#### Scenario: 根账户作为栏目标题
- **WHEN** 账户页面渲染
- **THEN** 每栏顶部显示根账户类型名称（资产、收入、支出、权益），根账户不可点击

#### Scenario: 账户卡片只显示名称
- **WHEN** 账户卡片渲染
- **THEN** 卡片上只显示账户名称，不显示余额或其他字段

### Requirement: 选中账户
系统 SHALL 支持点击账户卡片选中账户。反复点击已选中的卡片 SHALL 切换其子账户的展开/折叠状态，选中态保持不变。

#### Scenario: 点击账户卡片
- **WHEN** 用户点击一个未选中的账户卡片
- **THEN** 该账户被选中，从页面底部拉起抽屉显示账户详情和编辑控件

#### Scenario: 反复点击已选中的账户
- **WHEN** 用户点击当前已选中的账户卡片
- **THEN** 其子账户已展开则折叠，已折叠则展开，选中态和抽屉状态不变

#### Scenario: 点击已选中且无子账户的账户
- **WHEN** 用户点击当前已选中且没有子账户的账户卡片
- **THEN** 无任何展开/折叠变化，选中态保持不变

### Requirement: 展开子账户
系统 SHALL 在选中账户时展开其直接子账户，子账户以等宽卡片网格形式显示在父账户卡片下方，并采用包裹式纸片叠放高亮区分层级。

#### Scenario: 选中有子账户的账户
- **WHEN** 用户选中一个有子账户的账户
- **THEN** 该账户的直接子账户以等宽卡片网格形式展开显示在父账户下方
- **THEN** 每个展开层级使用 HSV 深度渐变背景，形成包裹式纸片叠放效果

#### Scenario: 选中无子账户的账户
- **WHEN** 用户选中一个没有子账户的账户
- **THEN** 不展开任何子账户网格

### Requirement: 选中与展开互斥
系统 SHALL 保证选中和展开的互斥逻辑：仅当新选中卡片不在当前展开路径 `expandedPath` 上时才折叠；新选中卡片在当前展开路径上时只切换选中态，不折叠子树。

#### Scenario: 选中父账户后选中子账户
- **WHEN** 用户先选中账户 A（展开子账户 B、C），再选中子账户 B（展开 D）
- **THEN** B 被选中并展开 D，A 不再被选中但不折叠（B 在 A 的子树内，折叠 A 会让 B 不可见）

#### Scenario: 同级账户切换
- **WHEN** 用户选中账户 B（展开 D），再选中同级账户 C
- **THEN** C 被选中，B 不再被选中且 D 被折叠（C 和 B 同级，折叠 B 不影响 C 可见性）

#### Scenario: 在展开路径上切换选中
- **WHEN** 当前展开路径为 [A, B, C]，用户点击路径上的 B
- **THEN** B 被选中，A 和 C 保持展开状态，C 的子树保持展开

#### Scenario: 点击外部取消选中
- **WHEN** 用户点击账户卡片以外的区域
- **THEN** 所有账户取消选中，所有子账户网格折叠，抽屉关闭

### Requirement: 抽屉式编辑面板
系统 SHALL 在选中账户时从页面底部拉起抽屉，显示账户详情和 inline 编辑控件。

#### Scenario: 抽屉显示账户信息
- **WHEN** 用户选中一个账户
- **THEN** 抽屉显示账户名称、账单日、还款日、所有者、关闭状态

#### Scenario: 编辑账户名称
- **WHEN** 用户在抽屉中修改账户名称并确认
- **THEN** 系统调用 rename API 更新账户名称

#### Scenario: 编辑账单日和还款日
- **WHEN** 用户在抽屉中修改账单日或还款日并确认
- **THEN** 系统更新对应字段

#### Scenario: 编辑账户所有者
- **WHEN** 用户在抽屉中修改账户所有者并确认
- **THEN** 系统调用 set_owner API 更新所有者

#### Scenario: 关闭账户
- **WHEN** 用户在抽屉中点击关闭账户
- **THEN** 系统调用 close API 关闭账户，卡片显示关闭状态

#### Scenario: 重开账户
- **WHEN** 用户在抽屉中对已关闭账户点击重开
- **THEN** 系统调用 open API 重开账户

#### Scenario: 删除账户
- **WHEN** 用户在抽屉中点击删除账户
- **THEN** 系统调用 delete API 删除账户（根账户和系统账户不可删除）

#### Scenario: 根账户不可编辑
- **WHEN** 用户选中的是根账户
- **THEN** 抽屉中不显示编辑控件，只显示账户信息

### Requirement: 新建账户入口
账户页 SHALL 提供新建账户按钮，点击后打开账户创建抽屉。新建账户必须基于当前选中的账户：未选中任何账户时新建按钮 MUST 处于禁用状态，不可交互。

#### Scenario: 显示新建按钮
- **WHEN** 用户在账户页
- **THEN** 页面提供新建账户按钮

#### Scenario: 选中账户后打开创建抽屉
- **WHEN** 用户已选中一个账户并点击新建账户按钮
- **THEN** 先折叠选中账户的编辑抽屉，再底部滑出创建抽屉，显示「将作为 <选中账户名> 的子账户创建」的提示，表单只包含名称等字段，不包含父账户和账户类型选择

#### Scenario: 未选中账户时新建按钮禁用
- **WHEN** 用户未选中任何账户
- **THEN** 新建账户按钮处于禁用状态，不可点击

### Requirement: 创建账户
系统 SHALL 支持通过抽屉创建新账户，新账户 SHALL 作为创建时选中账户的直接子账户，账户类型由父链自动推导，用户不可选择父账户或账户类型。创建后如需调整位置，通过拖放重定位完成。

#### Scenario: 填写创建信息
- **WHEN** 用户在创建抽屉中输入名称
- **THEN** 表单实时验证名称不为空

#### Scenario: 提交创建
- **WHEN** 用户点击确认
- **THEN** 系统调用 POST /api/accounts 并以当前选中账户为 parent_id 创建子账户，成功后刷新账户列表并关闭抽屉

#### Scenario: 创建失败
- **WHEN** API 返回错误（如名称重复）
- **THEN** 抽屉内显示错误信息

### Requirement: 拖放重定位账户
账户页 SHALL 支持拖放账户卡片改变其在账户树中的位置：将卡片拖动到另一张卡片上松手，被拖账户成为目标账户的子账户（整棵子树随之迁移）。移动成功后前端乐观更新账户树并调用 `PUT /api/accounts/{id}/parent`，失败时回滚并提示错误。已关闭账户 SHALL 同样可拖动。

#### Scenario: 拖到目标卡片上
- **WHEN** 用户将卡片 A 拖动到卡片 B 上松手
- **THEN** A 成为 B 的子账户，账户树乐观更新，系统调用移动 API

#### Scenario: 移动 API 失败
- **WHEN** 移动 API 返回错误（如同级重名）
- **THEN** 账户树回滚到拖动前状态，并显示错误提示

#### Scenario: 拖放中的视觉反馈
- **WHEN** 用户拖动卡片经过潜在目标卡片
- **THEN** 被拖卡片半透明跟随指针，命中的目标卡片高亮为放置目标

#### Scenario: 悬停自动展开
- **WHEN** 用户拖动卡片在未展开的父卡片上悬停片刻
- **THEN** 该父账户自动展开其直接子账户，允许继续向深层放置

#### Scenario: 根账户与系统账户不可拖动
- **WHEN** 用户尝试拖动根账户或系统账户
- **THEN** 不进入拖动态（根账户无卡片展示，系统账户卡片不响应拖动）

#### Scenario: 拖放完成后选中被拖账户
- **WHEN** 一次拖放完成（改父节点或同级排序）
- **THEN** 选中态切换到被拖放的账户卡片，效果如同拖放后点击了该卡片

### Requirement: 同级拖动排序
账户页 SHALL 支持将卡片拖动到同级卡片之间的位置调整同级顺序。同级顺序 SHALL 仅保存在浏览器 localStorage，不写入数据库、不跨设备同步；渲染时对每组同级账户按本地顺序排序，本地顺序中不存在的账户按创建顺序追加在尾部。

#### Scenario: 调整同级顺序
- **WHEN** 用户将卡片拖动到两个同级卡片之间松手
- **THEN** 同级顺序更新并写入 localStorage，不调用任何后端 API

#### Scenario: 刷新后顺序保持
- **WHEN** 用户刷新页面
- **THEN** 同级顺序按 localStorage 中保存的顺序渲染

### Requirement: 跨类型移动确认
系统 SHALL 在拖放目标与被拖账户的根账户类型不一致时，先弹出确认提示再执行移动，说明移动后账户及其子树的账户类型将改变。

#### Scenario: 跨类型放置需确认
- **WHEN** 用户将资产类账户卡片拖到支出类账户卡片上松手
- **THEN** 弹出确认提示「移动后该账户及其子账户将变为支出类型」，确认后才调用移动 API

#### Scenario: 取消跨类型移动
- **WHEN** 用户在确认提示中选择取消
- **THEN** 账户树保持原状，不调用移动 API
