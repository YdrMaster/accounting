# assets-visual-reports

## Purpose

资产页可视化报表——定义资产页面的 tab 布局、数据懒加载与周期控件，以及资产负债表 tab 的资产趋势折线图和现金流量 tab 的收支太阳图、资金流量表等可视化展示行为。

## Requirements

### Requirement: 资产页 Tab 布局
资产页 SHALL 提供「资产负债表」与「现金流量」两个 tab，默认显示资产负债表 tab，通过点击切换。

#### Scenario: 默认 tab
- **WHEN** 用户进入资产页
- **THEN** 显示资产负债表 tab 内容

#### Scenario: 切换 tab
- **WHEN** 用户点击「现金流量」tab
- **THEN** 显示现金流量内容，资产负债表内容隐藏

### Requirement: 数据懒加载
各 tab SHALL 在首次激活时才请求数据，未激活的 tab 不发起网络请求。

#### Scenario: 首次切入加载
- **WHEN** 用户首次切换到现金流量 tab
- **THEN** 系统请求 cash-flow 与 category-breakdown API

#### Scenario: 未切入不加载
- **WHEN** 用户停留在资产负债表 tab
- **THEN** 不请求 cash-flow 与 category-breakdown API

### Requirement: 周期控件
资产负债表 tab 顶部 SHALL 提供粒度下拉（周/月/年）；现金流量 tab 顶部 SHALL 提供周期前后导航（◀ ▶）与粒度下拉（周/月/年），两者共同驱动该 tab 内所有数据。

#### Scenario: 趋势图粒度切换
- **WHEN** 用户在资产负债表 tab 将粒度从月切换为年
- **THEN** 趋势图按年分桶重新请求并渲染

#### Scenario: 现金流量周期导航
- **WHEN** 用户在现金流量 tab 点击 ▶（当前为 2026-07，粒度为月）
- **THEN** 参考日期移至 2026-08，太阳图与流量表均重新加载 8 月数据

#### Scenario: 粒度影响导航步长
- **WHEN** 粒度为年，用户点击 ◀（当前参考日期 2026-07-15）
- **THEN** 参考日期移至 2025 年

### Requirement: 资产趋势折线图
资产负债表 tab SHALL 在现有汇总指标上方展示 ECharts 折线图：总负债为底部折线，净资产面积堆叠其上（顶边即总资产线），净资产区域以半透明色填充；tooltip 同时展示总资产、总负债、净资产三个值。

#### Scenario: 三线渲染
- **WHEN** 趋势 API 返回 points 数据
- **THEN** 图表渲染负债折线与净资产堆叠面积，视觉上呈现总资产线、总负债线及两者之间的填充区域

#### Scenario: tooltip 展示
- **WHEN** 用户悬停某个数据点
- **THEN** tooltip 显示该桶起始日及总资产、总负债、净资产三个金额

#### Scenario: 空数据
- **WHEN** 趋势 API 返回空 points
- **THEN** 图表区域显示空状态提示，不渲染坐标轴数据

### Requirement: 趋势图点数自适应
趋势图 SHALL 根据容器宽度决定显示点数：N = max(2, floor(容器宽度 / 48))，取全量数据的最近 N 个桶渲染；容器尺寸变化时重新切片，不重新请求 API。

#### Scenario: 窄屏截取
- **WHEN** 容器宽度 480px，API 返回 36 个月度点
- **THEN** 图表仅渲染最近 10 个点

#### Scenario: 窗口缩放
- **WHEN** 用户调整窗口宽度导致容器变宽
- **THEN** 图表显示更多历史点，无网络请求

### Requirement: 收支太阳图
现金流量 tab SHALL 展示收入与支出两个 ECharts 太阳图（sunburst），位于流量表上方；环层与账户层级一一对应（一环 = 一级账户，二环 = 二级账户，以此类推）。太阳图 SHALL 支持点击下钻：点击扇区时动画缩放至该节点并展开其子层级，点击中心返回上一级。扇区 SHALL 以圆角样式渲染；分类标签 SHALL 显示在扇区外侧并始终保持水平方向（不随扇区旋转），标签内容为分类名称与占周期总额百分比，过小扇区不显示标签；悬停时 tooltip 显示分类名称与金额。每一级 SHALL 仅显示超过该级总额 1% 的收支项；未超过的项不渲染为扇区、也不参与比例计算，可见项归一化后填满整环，以避免过窄的区域；中间分类的真实总额（含被过滤项）仍 SHALL 在悬停 tooltip 中显示。

#### Scenario: 层级映射
- **WHEN** 支出明细包含 "Expenses:餐饮:外卖" = 500 且 "Expenses:餐饮" 汇总 = 800
- **THEN** 一环「餐饮」扇区对应 800，二环「外卖」扇区对应 500，二环「餐饮」其余扇区（伪子节点）对应 300

#### Scenario: 中间层直接分录
- **WHEN** 某非叶子账户存在直接记入的分录（自身汇总 > 子节点汇总之和）
- **THEN** 插入与该账户同名的伪子节点承载差额，保证父环 = 子环之和

#### Scenario: 过滤微小项
- **WHEN** 支出根下「交通」汇总 5 元，占支出总额 1000 元的 0.5%
- **THEN** 一环不渲染「交通」扇区，可见分类按自身金额归一化填满整环

#### Scenario: 逐级过滤
- **WHEN** 「餐饮」汇总 800 元，其子类「奶茶」8 元（占餐饮 1%，未超过）
- **THEN** 下钻至「餐饮」时不渲染「奶茶」扇区，「餐饮」tooltip 仍显示真实总额 800 元

#### Scenario: 悬停提示
- **WHEN** 用户将指针悬停在支出太阳图的「餐饮」扇区上
- **THEN** tooltip 浮出显示分类名称「餐饮」与其金额

#### Scenario: 外侧水平标签
- **WHEN** 太阳图渲染完成且无指针悬停
- **THEN** 各扇区外侧通过短线连接显示水平方向的标签，内容为分类名称与百分比（如「餐饮 22.3%」），标签颜色与所属扇区一致

#### Scenario: 里圈标签白色描边
- **WHEN** 某节点不在最外环（其标签因 ECharts 限制只能渲染在扇区上，且颜色跟随扇区）
- **THEN** 该标签文字带白色描边，保证在扇区底色上可辨

#### Scenario: 过小扇区不显示标签
- **WHEN** 某扇区圆心角小于 5 度
- **THEN** 该扇区不显示外侧标签，悬停 tooltip 仍可用

#### Scenario: 点击下钻
- **WHEN** 用户点击支出太阳图的「餐饮」扇区
- **THEN** 图表以动画将该节点缩放至圆心，外环展示其下级子类

#### Scenario: 返回上级
- **WHEN** 用户在下钻状态点击太阳图中心
- **THEN** 图表以动画返回上一层级

### Requirement: 资金流量表
现金流量 tab SHALL 展示资金流量表：每行一个资产账户，列为流入、流出、净额，末行为合计；数据来自 `GET /api/reports/cash-flow`，金额以 decimal.js 格式化显示。

#### Scenario: 表格渲染
- **WHEN** cash-flow API 返回 items 与 total
- **THEN** 表格按账户逐行显示流入/流出/净额，末行显示合计

#### Scenario: 周期范围显示
- **WHEN** API 返回 period_start 与 period_end
- **THEN** 界面显示当前统计的周期范围
