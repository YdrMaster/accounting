## Context

当前主界面使用 `ResponsiveShell.vue` 作为布局容器，通过 `useResponsiveLayout.ts` 管理 4 个页面（calendar、budget、transaction、assets）的响应式排列。页面切换依赖 `WideHeader.vue` 中的左右箭头按钮和拖拽手势，移动端底部有圆点指示器。

后端已有完整的账户 CRUD API（list、create、rename、close、reopen、delete、set_owner），前端 `types/api.ts` 中缺少 `AccountDto` 类型定义。`AccountType` 枚举目前有 4 个变体（Asset、Equity、Income、Expense），缺少 Import 类型。

## Goals / Non-Goals

**Goals:**
- 新增账户管理页面，支持查看和编辑账户
- 用页面切换条替代固定标题，提供更直观的导航
- 桌面端和移动端差异化交互体验
- 后端支持 Import 类型的根账户

**Non-Goals:**
- 创建子账户功能（交互复杂，后续迭代）
- 账户余额展示（账户页面不显示余额）
- 账户页面的数据可视化（如图表）

## Decisions

### 1. 页面切换条组件设计

**决策**: 新建 `PageSwitcher.vue` 组件替代 `WideHeader.vue`，放在 `ResponsiveShell.vue` 顶部。

**理由**:
- 切换条需要感知当前可见页面数量和起始位置，与 `useResponsiveLayout` 紧密耦合
- 切换条本身需要支持拖动和点击，逻辑独立于内容区
- 桌面端左右箭头与切换条在同一横排，合并为一个组件更简洁

**替代方案**: 修改 `WideHeader.vue` 扩展功能。但 WideHeader 的 props 和事件模型与切换条差异较大，重构成本更高。

### 2. 账户页面布局

**决策**: 按根账户类型分 5 栏竖排（资产、收入、支出、权益、导入），每栏内账户以卡片网格展示。

**理由**:
- 与截图参考图的分类网格风格一致
- 竖排分栏让用户快速定位账户类型
- 根账户不可编辑，作为栏目标题展示

### 3. 选中/展开互斥逻辑

**决策**: 选中账户时展开其直接子账户行；切换选中时，只有当被取消选中的账户的子树中不包含新选中项时才折叠。

**理由**:
- 避免用户选中的子账户因父级折叠而不可见
- 同级切换时自动清理上一级的展开状态，保持界面整洁

**实现方式**: 维护 `selectedAccountId` 和 `expandedAccountIds` 两个状态。点击账户时：
1. 设置 `selectedAccountId` 为新账户
2. 将新账户的直接子账户 ID 加入 `expandedAccountIds`
3. 遍历旧的 `expandedAccountIds`，如果某个被展开账户的子树中不包含新选中账户，则从 `expandedAccountIds` 中移除

### 4. 抽屉式编辑面板

**决策**: 从页面底部滑入的抽屉（drawer），浮于卡片网格上方，显示选中账户的详情和编辑控件。

**理由**:
- 与移动端的交互习惯一致
- 不离开账户页面上下文，编辑完成后可继续浏览
- 浮层方式不破坏卡片网格布局

### 5. 桌面端内容区拖拽禁用

**决策**: 桌面端禁用内容区的拖拽页面切换，只通过切换条和左右箭头切换。

**理由**:
- 桌面端鼠标拖拽体验不如触摸直观
- 避免与页面内滚动冲突
- 切换条已提供足够的导航能力

## Risks / Trade-offs

- **[账户树性能]** 账户数量较多时，卡片网格和子账户展开可能影响渲染性能 → 账户数量通常有限（<100），虚拟滚动暂不需要
- **[抽屉与滚动冲突]** 抽屉浮层可能遮挡底部账户卡片 → 抽屉高度限制为页面的 50%，卡片网格区域预留足够空间
- **[切换条拖动与内容拖动的区分]** 移动端切换条和内容区都可拖动，需要明确触摸事件的归属 → 切换条和内容区是独立的 DOM 元素，事件自然隔离
